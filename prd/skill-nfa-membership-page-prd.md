# Skill NFA 会员机制与页面 PRD

- 文档版本：v0.1
- 创建日期：2026-03-15
- 文档目标：明确 `Skill` 的 NFA 会员机制、单个 premium 解锁机制，以及 `nfa/` 页面第一阶段的产品定义

## 1. 背景

社区已经基本接受一个双路径模型：

1. 单个 premium skill 解锁
2. NFA 会员长期通道

但当前不适合直接把这件事推进成“开卖”状态，原因有三个：

1. 需要先把真正有用的 skill 层持续补起来
2. 需要先把页面、入口、钱包连接和网络 readiness 打通
3. 需要先把会员机制讲清楚，避免用户把它误解为普通 NFT 发售

因此本阶段更适合做的是：

`先完成 NFA 机制定义 + 页面 preview + 钱包连接准备，不开启 live mint`

## 2. 产品目标

### 2.1 核心目标

1. 为 premium skill 建立长期承接结构
2. 给核心用户提供长期会员路径
3. 给普通用户保留低门槛单点进入方式
4. 为后续 NFA mint 和权益扩展提前铺好页面和交互基础

### 2.2 非目标

1. 本阶段不开放 mint
2. 本阶段不直接上线完整合约交互
3. 本阶段不做二级市场和复杂 NFT 玩法叙事
4. 本阶段不把 NFA 写成“投机资产”

## 3. 核心定位

`Skill NFA` 的定位不是普通头像 NFT，而是：

1. 长期会员身份凭证
2. premium skill 的高阶通行证
3. 后续权益层的承接容器
4. 面向长期支持者的身份入口

## 4. 双路径机制

### 4.1 路径 A：单个 premium 解锁

#### 定义

用户通过 burn `$10` 等值代币，获得某一个 premium skill 的访问权或解锁权。

#### 目的

1. 降低进入门槛
2. 让用户不用先买长期会员也能体验 premium 层
3. 验证哪些 premium skill 真实有需求

#### 适合用户

1. 只对某一个 skill 感兴趣的人
2. 想先体验再决定的人
3. 不愿意一次性承诺长期会员的人

### 4.2 路径 B：NFA 会员

#### 定义

用户通过 `0.1 BNB` 铸造一枚 `Skill NFA`，总量 `177`，持有即进入长期会员通道。

#### 目的

1. 沉淀长期支持者
2. 给 premium 层建立更稳定的会员结构
3. 给后续权益、身份、优先级系统提供承载容器

#### 适合用户

1. 愿意长期持有和跟随系统更新的人
2. 希望持续获得 premium skill 和优先权益的人
3. 希望拥有“早期支持者身份”的用户

## 5. 关键产品判断

### 5.1 为什么必须是双路径

如果只有单个 premium 解锁：

1. 系统会缺乏长期会员层
2. 用户关系会停留在一次性购买
3. 难以沉淀稳定社群身份

如果只有 NFA：

1. 进入门槛过高
2. 对只需要单个 skill 的用户不友好
3. 很容易变成“先卖身份，再想里面装什么”

因此双路径并存是必要条件。

### 5.2 为什么 NFA 页面先 preview 不开 mint

本阶段如果直接把按钮改成 live mint，会带来几个问题：

1. 页面先于机制成熟
2. 钱包连接、网络切换、后续交互仍需验证
3. premium skill 层还在持续补，权益承载还不够完整
4. 容易在社区里形成“先卖壳”的感知

因此当前页面状态必须是：

`preview live / mint closed`

## 6. NFA 机制设计

### 6.1 基础参数

| 项目 | 方案 |
| --- | --- |
| 总量 | 177 |
| 铸造价格 | 0.1 BNB |
| 单钱包限制 | 待定，建议 1 或 2 |
| 当前状态 | preview only |
| 第一阶段权益 | 会员身份 + 后续 premium 通道 |

### 6.2 第一阶段权益

建议第一阶段先聚焦这些可解释、可持续的权益：

1. premium skill 优先访问权
2. 新功能或新 skill 的优先体验权
3. 专属频道 / 专属讨论区资格
4. 未来活动、白名单、联名的优先级
5. 长期支持者身份展示

### 6.3 暂不承诺的权益

为了避免画饼，第一阶段不应承诺：

1. 固定空投收益
2. 保证型 financial utility
3. 过度复杂的链上游戏化
4. 尚未有产品基础支撑的高阶自治权

## 7. 页面目标

`nfa/` 页面第一阶段的目标，不是转化成交，而是把结构讲清楚、把连接能力准备好。

### 7.1 页面必须解决的问题

1. 让用户理解双路径模型
2. 让用户知道 NFA 不是普通 NFT
3. 让用户看到当前状态是 preview，不是 live mint
4. 让用户可以提前完成钱包连接和 BSC network readiness

### 7.2 页面不需要解决的问题

1. 不需要本阶段完成完整 mint 成交闭环
2. 不需要把所有未来权益都写死
3. 不需要在第一版就做复杂动态交互

## 8. 页面信息结构

### 8.1 Hero 区

需要讲清 3 件事：

1. 这是长期会员通道
2. 当前双路径结构是什么
3. 当前是 preview，mint 未开启

### 8.2 核心数字卡片

建议固定展示：

1. `177`
2. `0.1 BNB`
3. `$10 burn`

### 8.3 Wallet Panel

页面必须支持：

1. 检测 injected wallet
2. 显示当前连接钱包
3. 检测当前网络
4. 切换到 BSC Mainnet
5. 明确显示 `Mint Not Live Yet`

### 8.4 机制说明区

需要解释：

1. 单个 premium 解锁适合谁
2. NFA 会员适合谁
3. 为什么两条路要并存

### 8.5 当前状态说明区

需要明确写：

1. preview live
2. mint closed
3. 当前先解决页面、连接、结构问题

## 9. 钱包连接要求

### 9.1 支持目标

第一阶段默认兼容：

1. Binance Wallet
2. MetaMask
3. OKX Wallet

### 9.2 当前交互目标

1. `Connect Wallet`
2. `Switch to BSC`
3. 状态展示
4. 错误提示

### 9.3 当前不做的内容

1. 合约写操作
2. live mint transaction
3. claim / reveal / secondary market logic

## 10. 设计原则

### 10.1 语气原则

页面文案应避免：

1. 过度销售感
2. FOMO 式催促
3. 投机型叙事

更适合的表达是：

1. 结构清楚
2. 节奏克制
3. usefulness first
4. long-term lane

### 10.2 视觉原则

1. 不做普通 NFT marketplace 风格
2. 更接近 premium pass / protocol pass / membership card
3. 保持 SkillsHub 当前深色 + 金色体系
4. 明确区分 preview 和 live

## 11. 成功指标

第一阶段不看 mint 转化，主要看准备度：

1. 页面访问量
2. 钱包连接成功率
3. BSC network switch 成功率
4. 用户对双路径结构的理解反馈
5. 社区对 NFA 定位的接受度

## 12. 风险与应对

| 风险 | 描述 | 应对 |
| --- | --- | --- |
| 过早销售感 | 页面太像发售页 | 明确 preview / mint closed |
| 权益空心化 | 会员先于能力层成熟 | 继续优先补 skill 层 |
| 用户误解 | 把 NFA 当普通图片 NFT | 文案强调 membership lane |
| 连接问题 | 钱包和网络切换体验差 | 先做连接与网络 readiness |

## 13. 下一步执行顺序

建议按这个顺序推进：

1. 持续补真正有用的 skill
2. 把 `nfa/` 页面继续打磨成稳定 preview 入口
3. 明确 premium skill 首批名单
4. 确定 NFA 第一阶段权益边界
5. 最后再进入 mint 设计与合约阶段

## 14. 一句话总结

`Skill NFA` 当前最合理的推进方式，不是马上开卖，而是先作为长期会员路径的 preview 入口立起来；与此同时，用持续可用的 skill 更新，把这个会员体系真正撑起来。
